# AnonWallet

A privacy-preserving testnet Ethereum wallet that generates a fresh address for every transaction, breaking the on-chain links that normally tie a user's activity together.

> **Testnet only.** AnonWallet runs on Ethereum testnets (Sepolia) and is built as a learning and portfolio project. It is not intended for mainnet or real funds.

---

## Why AnonWallet

Most wallets reuse a single public address, which makes a user's entire transaction history trivially linkable on a block explorer. AnonWallet takes a different approach: it derives a new address for each transaction from a single seed phrase, so activity is far harder to correlate. The seed never leaves the browser and is encrypted at rest.

## Features

- **Hierarchical deterministic wallet** — BIP-44 derivation generates unlimited addresses from one seed phrase.
- **Fresh address per transaction** — reduces on-chain linkability between transfers.
- **Client-side encryption** — the seed phrase is encrypted locally with a password (PBKDF2 key derivation + XSalsa20-Poly1305). Nothing is sent to a server.
- **Lock / unlock flow** — the wallet locks to a password-protected state; unlocking decrypts the stored seed.
- **Send, Receive, and History** — send testnet ETH, display a QR code to receive, and review past transactions with status filters.
- **Network switching** — switch between supported testnets.
- **Settings** — export the seed phrase (password-gated) and change the encryption password.

## Tech Stack

- **Framework:** Next.js (App Router) + TypeScript
- **Styling:** Tailwind CSS
- **Icons:** lucide-react
- **Blockchain:** ethers.js v6
- **Encryption:** tweetnacl (XSalsa20-Poly1305) with PBKDF2 key derivation

## Getting Started

### Prerequisites

- Node.js 18 or later
- npm

### Installation

```bash
git clone https://github.com/Tobby004/privacy-testnet-wallet.git
cd privacy-testnet-wallet
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Getting testnet ETH

To try sending transactions you'll need testnet ETH. Faucet availability changes often, but commonly working options include the Chainlink, QuickNode, and Alchemy Sepolia faucets. Copy an address from the Addresses page and request funds.

## How It Works

### Address derivation

A 12-word seed phrase is generated (or imported), then addresses are derived along the BIP-44 path `m/44'/60'/0'/0/{index}`. Incrementing the index produces a new, unlinked address while remaining fully recoverable from the original seed.

### Key storage

When a wallet is created, the seed phrase is encrypted with a key derived from the user's password and stored in the browser's local storage. The plaintext seed only exists in memory while the wallet is unlocked.
